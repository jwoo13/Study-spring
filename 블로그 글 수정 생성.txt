보통 설계하면
1.모든 글 조회
2.글의 내용
3.삭제
4.수정 추가 동시에함

글을 생성할 때는 URL에 별도 쿼리 파라미터가 x
사용자 -> 뷰 컨트롤 -> 뷰
(생성)/new-article

하지만 수정할때는 URL에 ?id=123과 같이 수정할 글의 id를 쿼리 파라미터에 추가 요청해야함
/new-article?id=123 -> 뷰 컨트롤러 -> 뷰
(id가 123인 글 수정)			123id를 가진 엔티티 조회 후 모델에 추가


여기서 쿼리 파라미터란 HTTP 요청에서 URL의 끝에 '?'로 시작하는 키 값으로 이루어진 문자열이며 &로 구분함
예를 들어 ?id =123의 때에는 키는 id 값은 123이 됨
즉 쿼리 파라미터가 있느 경우 컨트롤러 메소드는 수정을 해야 하므로 엔티티를 조회해 기존 글 데이터를 모델에 넣어 화면에 보여줘야함

쿼리 파라미터가 없는 때에는 새 글 이므러 화면에 아무것도 보여줄 필요 x
또한 뷰에서는 쿼리 파라미터의 id여부에 따라 다른 수정과 생성중 적절한 버튼 보여줘야함
